# 🪖 RTS 3D 美术生产指南（Low-Poly 半写实风）

此文档为 Web 2.5D RTS 项目定制，提供给 3D 建模师和贴图师直接开工的生产说明书。所有资产必须保持风格统一（低多边形、半写实军事风），并且为程序中的性能优化和阵营换色做好结构准备。

---

## 1. 步兵 (Infantry)

![Infantry](C:\Users\16762\.gemini\antigravity\brain\b217c776-a6a0-4e80-8b6c-19d15be66f87\lp_infantry_1776344264725.png)

### 整体外观设计
持无托/紧凑型突击步枪的现代/近现代步兵。体态需要稍微夸张（头胸肩略大），以保证在斜俯视角下能看到肩背动作。模型切忌碎末零件，去掉水壶、微型胸挂等细节。

### 生产方案
- **配色方案**：主体为哑光军绿/暗灰，头盔、护肩、或者大面积的防弹衣区域留作**阵营换色区**。
- **材质方案**：单一 PBR PBR材质（StandardMaterial），粗糙度（Roughness）设定在 0.7-0.9。不要做金属反射。
- **建模结构建议**： 
  - 面数控制（Tris）：$< 1200$
  - 骨骼：提供基础 Humanoid 绑定。
  - 特点：**枪械必须粗大**（夸张 1.5 倍），否则俯视角看不见。手脚末端可以做成连体块，不需要分指。
- **贴图方案**：尺寸控制在 $256 \times 256$ ，采用色块拼合或极简的光影烘焙（AO）。
- **阵营换色区域**：**头盔、肩甲、防弹衣背心**。在材质球中将这部分分离出一个独立材质命名为 `TeamColor_Mat`，以方便代码端直接替换 Diffuse Color。

## 2. 轻型坦克 (Light Tank)

![Light Tank](C:\Users\16762\.gemini\antigravity\brain\b217c776-a6a0-4e80-8b6c-19d15be66f87\lp_tank_1776344280532.png)

### 整体外观设计
轮廓呈明确的矩形+八角形炮塔，炮管粗且短，底盘略宽于炮塔。采用硬切角（Hard Edges），无需倒角。

### 生产方案
- **配色方案**：主体为沙地色/军绿色，车身侧面中端和炮塔后脑勺区域使用高亮的阵营色涂装。
- **材质方案**：车身为哑光烤漆（Roughness $0.5-0.6$），履带使用极简的金属深色。
- **建模结构建议**：
  - 面数控制（Tris）：$< 1500$
  - 层级结构：**必须分件**。要求分为 `Tank_Body` (车身基座) 和 `Tank_Turret` (可随鼠标旋转的炮塔)。履带不用做单片独立结构，做成一个环状整体即可。
- **贴图方案**：UV可复用（左右对称）。如果需要履带滚动动画，履带部分贴图沿 V 轴能产生无缝拉伸。
- **阵营换色区域**：炮塔两侧的装甲板、车身上的明显条带。这部分使用分离材质 `TeamColor_Mat`。

## 3. 主基地 (Main Base)

![Main Base](C:\Users\16762\.gemini\antigravity\brain\b217c776-a6a0-4e80-8b6c-19d15be66f87\lp_base_1776344352788.png)

### 整体外观设计
一个极具厚重感、占地面积大的工业指挥中心。呈金字塔型或厚重的台阶状。要有明显的“入口”或者“指挥雷达”，作为俯视角的视线焦点。

### 生产方案
- **配色方案**：以混凝土灰、特种钢材暗黑为主。楼顶区域或基地四个边角，铺设大面积的阵营原色。
- **材质方案**：需要混合材质，主建筑质感粗糙（建筑外墙），玻璃窗使用纯黑色或自发光（Emission）黄色，阵营色面板为烤漆材质。
- **建模结构建议**：
  - 面数控制（Tris）：$< 2000$ 
  - 结构：底部必须尽量贴合在平整的正方形/六边形网格内（以便在场景格子上摆放）。建筑不留悬空部分。
- **贴图方案**：$512 \times 512$ 色板。可以在建筑基座边缘画一点泥土色以过渡地形。
- **阵营换色区域**：**主雷达罩、屋顶的大面积漆面、基地墙体的结构带**。使用 `TeamColor_Mat` 命名。

## 4. 资源点 (Resource Node)

![Resource Node](C:\Users\16762\.gemini\antigravity\brain\b217c776-a6a0-4e80-8b6c-19d15be66f87\lp_resource_1776344368982.png)

### 整体外观设计
设计为一簇从龟裂地面突出的发光矿石/人造补给堆。轮廓必须呈现出“可被打破”或“可被提取”的尖锐结构。

### 生产方案
- **配色方案**：与地面有强烈对比度。例如赤红色平地配上晶蓝色的矿石。
- **材质方案**：矿物部分开启自发光（Emissive），并调低粗糙度制作出玻璃晶体质感。
- **建模结构建议**：
  - 面数控制（Tris）：$< 500$
  - 特点：模型底部制作出一圈隆起的“陨石坑/龟裂基座”，以便在任何地形上摆放都不显得突兀。
- **可采集感的视觉强化方式**：留出一个骨骼或者使用 Shader 制作极慢的上下浮动/呼吸灯缩放。
- **贴图方案**：可以使用顶点色（Vertex Color）或者极简渐变贴图即可，资源点以强剪影为主。

---
> [!IMPORTANT]  
> ### 🔌 引擎接入规范（PlayCanvas / Web RTS）
> 1. 所以模型一律导出为 `*.glb` 格式，合并形变（Apply Modifiers）。
> 2. 轴向：Y 轴朝上（Y-Up），Z 轴为正前方面朝向（+Z Forward）。这极其重要！
> 3. 所有需要随属于玩家颜色变换的 Mesh 面，其 Material 的名字必须统一命名为 **`TeamColor_Mat`**。我们在代码端（`TeamColorManager`）会对其 Diffuse 通道进行劫持与动态覆写。
